COMO HACER RAMAS NUEVAS Y HACER MERGES

1-Para crear una rama nueva hacemos "git branch <nombre rama>"

2-Para cambiarnos a la rama nueva hacemos "git switch <rama a la que queremos cambiarnos>"
	2-1-Tambien se puede usar git checkout <nombre-rama>

3-Una vez estemos en la nueva rama, podemos hacer modificaciones a los archivos locales, aunque hay que tener
en cuenta que en el repositorio online no hay nada, por lo cual hay que hacer un push de todo o de todo lo que queramos subir

4-Ahora tenemos todo subido a nuestra rama.

5-Para hacer un merge, (Ojo el merge se hace de forma local por lo cual en el repositorio remoto no veremos nada cuando hagamos el merge)
hacemos vamos a nuestra rama que queremos que reciba el merge, es decir, si tengo una rama test y quiero unirla con master,
hago git switch master, y luego hago git merge test, esto me va a hacer el merge localmente, para subirlo a el repo online hacemos un push y listo.

PD: Todo esto se hace suponiendo que hemos hecho un git pull, y justo despues de hacer un git init, hacemos git remote add origin "repo url"

EXTRAS
Si has borrado un archivo local pero esta subido a github, puedes hacer git restore <filename> y te los restablece
